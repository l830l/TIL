##### 이진 트리 표현하기
###### 전위 순회 과정 살펴보기
- 전위 순회는 현재 노드를 부모 노드로 생각했을 때 부모 노드 > 왼쪽 자식 노드 > 오른쪽 자식 노드 순서로 방문한다.
- 거치는 노드를 우선 방문하므로 직관적으로 이해하기 쉬우며 트리를 복사할 때 많이 사용된다.

```
		  [1]
		  / \
		 4	 8
	  / \   \
	 3   5   7
  /       /
 2       6
```
1. 루트 노드에서 시작한다. 현재 노드를 부모로 생각하면  방문 순서는 1 > 4 > 8 이다. 그러니 현재 자신의 노드 1을 방문한 다음 왼쪽 자식인 노드 4로 이동한다.

```
		  [1]
		  / \
		[4]	 8
	  / \   \
	 3   5   7
  /       /
 2       6
```
2. 노드 4로 이동하여 방문 순서는 4 > 3 > 5 이므로 자신을 방문하고 3으로 이동한다.
```
		  [1]
		  / \
		[4]	 8
	  / \   \
	[3]  5   7
  /       /
 2       6
```
3. 노드 3으로 이동하여 방문 순서는 3 > 2 > (없음) 이므로 자신을 방문하고 2로 이동한다.
```
		  [1]
		  / \
		[4]	 8
	  / \   \
	[3]  5   7
  /       /
[2]      6
```
4. 노드 2로 이동하여 2 > (왼쪽 없음) > (오른쪽 없음) 이므로 자신을 방문하고 3으로 거슬러 올라간다. 그런데, 노드 3은 자신과 왼쪽 자식은 방문했고 더 방문할 자식이 없으므로 4로 거슬러 올라간다. 노드 4는 4 > 3 > 5 이므로 5로 이동한다.
```

		  / \
		[4]	 8
	  / \   \
	[3] [5]  7
  /       /
[2]      6

```

##### 이진 트리 탐색하기
- 이진 트리에서 가장 중요한 것은 바로 탐색을 효율적으로 할 수 있도록 트리를 구축하는 것이다. 

###### 이진 탐색 트리 구축하기
- 이진 탐색 트리의 대상 데이터가 3 > 4 > 2 > 8 > 9 > 7 > 1 순서로 들어온다고 생각하고 이진 탐색 트리부터 구축한다.
- 이진 탐색 트리는 데이터 크기를 따져 크기가 작으면 왼쪽 자식 위치에, 크거나 같으면 오른쪽 자식 위치에 배치하는 독특한 정렬 방식을 갖고 있다. 
- 데이터를 하나씩 삽입하면서 삽입과 동시에 정렬을 한다. 
- 과정
	1. 최초의 데이터는 루트 노드가 된다. 3을 이진 탐색 트리에 루트 노드로 추가한다.

	2. 삽입하려는 4는 3보다 크므로 오른쪽 자식 위치에 배치한다.
		![[Pasted image 20260530212248.png]]
	3. 삽입하려는 2는 3보다 작으므로 왼쪽 자식 위치에 삽입한다.
		![[Pasted image 20260530212407.png]]

###### 이진 탐색 트리 탐색하기
- 방법
	- 찾으려는 값과 현재 노드를 비교한다.
		- 현재 노드의 값과 같으면 탐색을 종료한다.
		- 현재 노드의 값보다 크면 오른쪽 노드를 탐색한다.
		- 현재 노드의 값보다 작으면 왼쪽 노드를 탐색한다.
	- 값을 찾으면 종료한다.
	- 노드가 없을 때까지 계속 탐색했는데 값이 없으면 현재 트리에 값이 없는 것이다. 
- 배열 탐색과의 비교
	- 5를 찾으려고 할 때, 배열은 순차적으로 앞에서부터 탐색한다. 아래 배열에서 5가 없다는 것을 알아내기 위하여 7번의 비교 연산을 하였다. 그에 반해 이진 탐색 트리는 단 2번만에 비교 연산을 진행하며 알아낼 수 있다. 
![[Pasted image 20260530215710.png]]

- 이런 점에서 배열 탐색보다 이진 탐색 트리의 탐색이 훨씬 빠르다고 볼 수 있다. 
- 모든 탐색 알고리즘에서 탐색 효율을 개선하는 방법은 같다. 탐색 대상이 아닌 노드를 한 번에 많이 제외할 수 있으면 된다. 
- 탐색 트리의 구축 방식 자체가 갖는 특성은 데이터 크기에 따라 하위 데이터 중 한 방향을 검색 대상에서 제외하므로 검색을 빠르게 만들어준다. 

###### 이진 탐색 트리의 시간 복잡도
- 이진 탐색 트리의 시간 복잡도는 트리 균형에 의존한다. 
- 트리의 균형이 잡혔다는 것은 각 노드의 차수가 비슷하게 유지되면서 각 노드의 자식 노드 수가 비슷하게 유지되는 것을 뜻한다.
	- 왼쪽과 오른쪽 서브 트리의 높이 차가 1 이하인 경우를 말한다.
- 균형이 유지되었다고 가정했을 때, 삽입과 탐색 연산 시 시간 복잡도는 O(log N)

###### 이진 탐색 트리와 배열 탐색의 효율 비교
- 치우쳐진 형태의 트리
	![[Pasted image 20260530225057.png]]
	- 이진트리에서 오른쪽 자식만 있을 때, 9를 찾는다면 모든 노드를 탐색해야 하므로 O(N)의 시간 복잡도가 필요하다. 이 때는 배열과 같다. 
	- 하지만 사실 대부분의 상황에서는 이진 탐색 트리의 탐색 성능이 더 좋다. 
	- 균형 이진 탐색 트리(balanced binary search tree)
		- 치우쳐지지 않도록 균형을 유지하는 트리
		- 세부적으로 AVL 트리, 레드-블랙 트리 등으로 구분하여 부른다.
