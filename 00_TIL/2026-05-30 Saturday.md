##### 이진 트리 표현하기
###### 전위 순회 과정 살펴보기
- 전위 순회는 현재 노드를 부모 노드로 생각했을 때 부모 노드 > 왼쪽 자식 노드 > 오른쪽 자식 노드 순서로 방문한다.
- 거치는 노드를 우선 방문하므로 직관적으로 이해하기 쉬우며 트리를 복사할 때 많이 사용된다.

```
		  [1]
		  / \
		 4	 8
	  / \   \
	 3   5   7
  /       /
 2       6
```
1. 루트 노드에서 시작한다. 현재 노드를 부모로 생각하면  방문 순서는 1 > 4 > 8 이다. 그러니 현재 자신의 노드 1을 방문한 다음 왼쪽 자식인 노드 4로 이동한다.

```
		  [1]
		  / \
		[4]	 8
	  / \   \
	 3   5   7
  /       /
 2       6
```
2. 노드 4로 이동하여 방문 순서는 4 > 3 > 5 이므로 자신을 방문하고 3으로 이동한다.
```
		  [1]
		  / \
		[4]	 8
	  / \   \
	[3]  5   7
  /       /
 2       6
```
3. 노드 3으로 이동하여 방문 순서는 3 > 2 > (없음) 이므로 자신을 방문하고 2로 이동한다.
```
		  [1]
		  / \
		[4]	 8
	  / \   \
	[3]  5   7
  /       /
[2]      6
```
4. 노드 2로 이동하여 2 > (왼쪽 없음) > (오른쪽 없음) 이므로 자신을 방문하고 3으로 거슬러 올라간다. 그런데, 노드 3은 자신과 왼쪽 자식은 방문했고 더 방문할 자식이 없으므로 4로 거슬러 올라간다. 노드 4는 4 > 3 > 5 이므로 5로 이동한다.
```
		  [1]
		  / \
		[4]	 8
	  / \   \
	[3] [5]  7
  /       /
[2]      6
```
5. 노드 5는 본인을 방문하고 방문할 자식이 없으므로 노드 4로 거슬러 올라간다. 노드 4도 더 이상 방문할 자식이 없으므로 노드 1로 올라간다. 노드 1은 1 > 4 > 8 이므로 8 로 이동한다. 

```
		  [1]
		  / \
		[4]	[8]
	  / \   \
	[3] [5]  7
  /       /
[2]      6
```
6. 이런 방식으로 출력하면 1 > 4 > 3 > 2 > 5 > 8 > 7 > 6 순으로 순회하게 된다.
```
		  [1]
		  / \
		[4]	[8]
	  / \   \
	[3] [5] [7]
  /       /
[2]     [6]
```

###### 중위 순회 과정 살펴보기
- 현재 노드를 부모 노드로 생각했을 때 방문 우선순위가 왼쪽 자식 노드 > 부모 노드 > 오른쪽 자식 노드이다.
	- 거쳐가는 현재 노드를 거치기만 할 뿐 '방문'하지는 않는다.
		- 방문 : 탐색을 마친 상태로 탐색 과정에서 지나치는 것과 그렇지 않은 것을 구분하기 위해 방문이라는 용어를 사용한다.

- 노드를 삭제할 때 부모를 먼저 삭제하면 안된다. 자식부터 삭제해야 트리를 유지하며 재귀로 루트 노드까지 삭제가 가능하기 때문이다. 그래서 자식 노드부터 방문하는 후위 순위는 트리 삭제에 자주 활용된다.
