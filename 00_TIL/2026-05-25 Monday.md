###### 문제 15 표 편집
- [문제 링크](school.programmers.co.kr/learn/courses/30/lessons/81303)
- 문제 분석
	- 행 갯수 n, 처음 위치 k
	- U X, D X : 선택 위치 에서 U,D 로 X만큼 이동
	- C : 삭제후 아래로 이동
	- Z : 복구. 단 선택 행은 바뀌지 않음
	- 위에서부터 기본 O, 삭제 X 로 표현
- 의사코드
	1. TreeSet 을 만들고, 표의 행만큼 숫자들을 넣어준다.
	2. StringBuilder 를 만들고 표의 행만큼 ‘O’ 로 채워준다.
	3. 명령어에 따라 selected 를 움직이거나 삭제 및 복구한다.
	    1. U N 이면 current 를 selected 로 지정한다음 N만큼 lower 를 찾고 selected 를 재지정한다.
	    2. D N 이면 current 를 selected 로 지정한다음 N만큼 higher 를 찾고 selected 를 재지정한다.
	    3. C 이면 deletedN 에 삭제 번호를 넣고, selected 를 마지막이면 lower 로, 아니면 higher 로 재지정한 뒤, 삭제 번호를 스택에 넣고 TreeSet 에서 제거한다.
	    4. Z 이면 set 에 stack 맨 위 번호를 뽑아서 다시 넣어준다.
	4. 명령어가 끝나면 deletedStack 에 있는 번호들에 해당하는 곳만 ‘X’ 로 바꿔준다.
- 푼 코드
``` java
class Solution {  
    public String solution(int n, int k, String[] cmd){  
        TreeSet<Integer> set = new TreeSet<>();  
        StringBuilder sb = new StringBuilder();  
        for(int i = 0; i < n; i++){  
            set.add(i);  
            sb.append('O');  
        }  
        Stack<Integer> deletedStack = new Stack<>();  
        int selected = k;  
  
        for(String command: cmd){  
            switch(command.charAt(0)){  
                case 'U' -> {  
                    int upN = Integer.parseInt(command.split(" ")[1]);  
                    int current = selected;  
                    for(int i = 0; i < upN; i++){  
                        current = set.lower(current);  
                    }  
                    selected = current;  
                }  
                case 'D' -> {  
                    int downN = Integer.parseInt(command.split(" ")[1]);  
                    int current = selected;  
                    for(int i = 0; i < downN; i++){  
                        current = set.higher(current);  
                    }  
                    selected = current;  
                }  
                case 'C' -> {  
                    int deleteN = selected;  
  
                    if(selected == set.last()){  
                        selected = set.lower(set.last());  
                    }else{  
                        selected = set.higher(selected);  
                    }  
                    deletedStack.push(deleteN);  
                    set.remove(deleteN);  
                }  
                case 'Z' -> {  
                    set.add(deletedStack.pop());  
                }  
            }  
        }  
  
        while(!deletedStack.isEmpty()){  
            sb.setCharAt(deletedStack.pop(), 'X');  
        }  
  
        return sb.toString();  
    }  
}
```

``` java
class Row{  
    int up;  
    int down;  
    Row(int up, int down){  
        this.up = up;  
        this.down = down;  
    }  
    @Override  
    public String toString() {  
        return  
                "up=" + up +  
                ", down=" + down + "\n";  
    }  
}  
class Deleted{  
    int num;  
    Row row;  
  
    Deleted(int num, Row row){  
        this.num = num;  
        this.row = row;  
    }  
}  
class Solution {  
    public String solution(int n, int k, String[] cmd){  
        // 표의 각 행을 위 아래를 가르키는 것으로 만들되, 가상공간을 도입하여 맨 위 맨 아래 하나씩 더 둔다.  
        Map<Integer, Row> table = new HashMap<>();  
        table.put(-1, new Row(-2, 0));  
        StringBuilder answer = new StringBuilder();  
        for(int i = 0; i < n; i++){  
            table.put(i, new Row(i - 1, i + 1));  
            answer.append('O');  
        }  
        table.put(n, new Row(n - 1, n + 1));  
  
        Stack<Deleted> deletedStack = new Stack<>();  
  
        // 현재 선택 중인 행을 선택할 것을 선택  
        int selected = k;  
  
        // 명령어만큼 반복  
        for(String command: cmd){  
            switch(command.charAt(0)){  
                case 'U' -> {  
                    int upN = Integer.parseInt(command.split(" ")[1]);  
                    for(int i = 0; i < upN; i++){  
                        selected = table.get(selected).up;  
                    }  
                }  
                case 'D' -> {  
                    int downN = Integer.parseInt(command.split(" ")[1]);  
                    for(int i = 0; i < downN; i++){  
                        selected = table.get(selected).down;  
                    }  
                }  
                case 'C' -> {  
                    // 삭제할 번호를 저장한다.  
                    int deleteNum = selected;  
                    // 먼저 위 아래 행을 바꿔준다.  
                    table.get(table.get(selected).up).down = table.get(selected).down;  
                    table.get(table.get(selected).down).up = table.get(selected).up;  
  
                    // 해당 수를 스택에 넣어준 뒤 테이블에서 삭제한다.  
                    deletedStack.push(new Deleted(selected, table.get(selected)));  
  
                    if(table.containsKey(table.get(table.get(selected).down).down)){  
                        selected = table.get(selected).down;  
                    }else{  
                        selected = table.get(selected).up;  
                    }  
  
                    table.remove(deleteNum);  
                }  
                case 'Z' -> {  
                    // 스택에 있는 것을 뽑아 다시 넣어준다.  
                    Deleted deleted = deletedStack.pop();  
                    table.put(deleted.num, deleted.row);  
  
                    // 위 아래도 복구  
                    table.get(table.get(deleted.num).up).down = deleted.num;  
                    table.get(table.get(deleted.num).down).up = deleted.num;  
                }  
            }  
        }  
  
        while(!deletedStack.isEmpty()){  
            answer.setCharAt(deletedStack.pop().num, 'X');  
        }  
          
        return answer.toString();  
    }  
}
```

##### 몸 풀기 문제
###### 문제 15 요세푸스 문제
- 문제 설명
	- N명의 사람이 원 형태로 서 있습니다.
	- 각 사람은 `1번`부터 `N번`까지 번호표를 가지고 있습니다.
	- 임의의 숫자 `K`가 주어졌을 때, 다음 규칙에 따라 사람을 제거합니다.
		1. `1번` 사람을 기준으로 `K번째` 사람을 제거합니다.
		2. 제거된 사람의 다음 사람을 기준으로 다시 `K번째` 사람을 제거합니다.
		3. 이 과정을 반복합니다.
		4. 마지막에 살아남은 사람의 번호를 반환합니다.
	-  제약 조건
		- `N`과 `K`는 1 이상 1000 이하의 자연수입니다.
	- 입출력 예
		-  N, K, return  => 5, 2, 3
- 의사 코드
	1. 큐를 선언하고 모든 숫자를 넣는다.
	2. k 전까지 큐에서 빼서 다시 넣고 k 번째는 삭제한다.
	3. 이것이 큐가 1개만 남을 때까지 반복한다.
	4.  마지막 남은 것을 반환한다.
- 푼 코드
``` java
class Solution {  
    public int solution(int n, int k){  
        // 1. 큐를 선언하고 모든 숫자를 넣는다.  
        Queue<Integer> queue = new LinkedList<>();  
        for(int i=1; i<=n; i++){  
            queue.add(i);  
        }  
        // 2.k 전까지 큐에서 빼서 다시 넣고 k 번째는 삭제한다.  
        // 3. 이것이 큐가 1개만 남을 때까지 반복한다.  
        while(true){  
            if(queue.size() == 1) break;  
            for(int i=1; i <= k; i++){  
                if(i == k){  
                    queue.poll();  
                }else{  
                    queue.add(queue.poll());  
                }  
            }  
        }  
  
        // 4. 마지막 남은 것을 반환한다.  
        return queue.poll();  
    }  
}
```
- 참고 자료
	- `ArrayDeque`는 내부적으로 **원형 배열(circular array)** 기반 구현
	- Deque : 양쪽 끝에서 넣고 뺄 수 있는 큐
		- addLast(= add) / offerLast => 뒤에 넣기
		- addFirst => 앞에 넣기
		- pollFirst : 앞에서 꺼내기 + 제거
		- pollLast: 뒤에서 꺼내기 + 제거

##### 합격자가 되는 모의 테스트
###### 문제 16 기능 개발
- [문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/42586)
- 문제 분석
	- 작업 진행도 배열, 하루마다 진행되는 개발 속도 배열
	- 앞의 기능이 배포될 때 함께 배포됨
	- 각 배포마다 몇 개의 기능이 배포되는지 리턴
- 의사 코드
	1. 큐를 만든다.
	2. 각 진행률과 개발 속도를 클래스로 묶는다.
	3. 하루가 지나면 진행률을 전부 업데이트 해준다.
	4. 앞에서 부터 진행률이 100%인 것을 꺼내서 세는 것을 반복하고 100% 미만이 있다면 멈춘다. 
	5. num 이 0 이면 넣지 말고 다시 하루를 진행한다. num 이 1이상이면 응답 answer 에 넣어준다.
	6. 이것을 큐가 빌 때까지 반복한다.
- 푼 코드
``` java
class Work{  
    int progress;  
    int speed;  
  
    Work(int progress, int speed){  
        this.progress = progress;  
        this.speed = speed;  
    }  
  
    void updateProgress(){  
        this.progress += this.speed;  
    }  
}  
class Solution {  
    public int[] solution(int[] progresses, int[] speeds) {  
        ArrayList<Integer> answer = new ArrayList<>();  
        // 1. 큐를 만든다.  
        // 2. 각 진행률과 개발 속도를 클래스로 묶는다.  
        Queue<Work> workList = new ArrayDeque<>();  
        int len = progresses.length;  
        for(int i = 0; i < len; i++){  
            workList.add(new Work(progresses[i], speeds[i]));  
        }  
  
        // 6. 이것을 큐가 빌 때까지 반복한다.  
        while(!workList.isEmpty()){  
            int num = 0;  
  
            //3. 하루가 지나면 진행률을 전부 업데이트 해준다.  
            workList.forEach(Work::updateProgress);  
  
            //4. 앞에서 부터 진행률이 100%인 것을 꺼내서 세는 것을 반복하고 100% 미만이 있다면 멈춘다.  
            while(true){  
                if(workList.isEmpty() || workList.peek().progress < 100) break;  
                workList.poll();  
                num ++;  
            }  
            // 5. num 이 0 이면 넣지 말고 다시 하루를 진행한다. num 이 1이상이면 응답 answer 에 넣어준다.  
            if(num > 0) answer.add(num);  
        }  
  
        return answer.stream().mapToInt(i -> i).toArray();  
    }  
}
```

###### 문제 17 카드 뭉치
- [문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/159994)
- 의사 코드
	- 첫 번째 카드 뭉치 단어를 넣을 큐를 만든다.
	- 두 번째 카드 뭉치 단어를 넣을 큐를 만든다.
	- 원하는 배열의 첫번째 부터 반복해서 큐의 첫번째 단어와 일치하는 것이 있으면 뽑는다.
	- 큐 맨 앞의 것이 일치하는 것이 없으면 반복문을 멈추고 No를 반환한다.
	- 반복문이 끝까지 돈다면 Yes 를 반환한다.
- 푼 코드
``` java
import java.util.*;

class Solution {
    public String solution(String[] cards1, String[] cards2, String[] goal) {
        Queue<String> cardQ1 = new ArrayDeque<>();
        Queue<String> cardQ2 = new ArrayDeque<>();
        Arrays.stream(cards1).forEach(cardQ1::offer);
        Arrays.stream(cards2).forEach(cardQ2::offer);
        for(String findWord : goal){
            if(!cardQ1.isEmpty() && findWord.equals(cardQ1.peek())){
                cardQ1.poll();
                continue;
            }
            if(!cardQ2.isEmpty() && findWord.equals(cardQ2.peek())){
                cardQ2.poll();
                continue;
            }
            return "No";
        }

        return "Yes";
    }
}
```

#### 08 해시
##### 해시의 개념
- 해시는 해시 함수를 사용해서 변환한 값을 인덱스로 삼아 키와 값을 저장해 빠른 데이터 탐색을 제공하는 자료구조
- 해시는 키를 사용해 데이터 탐색을 빠르게 한다.

###### 해시의 특징
- 단방향으로 동작 : 키를 통해 값을 찾을 수 있지만, 값을 통해 키를 찾을 수는 없다.
	- 단방향으로만 동작하는 해시의 특성은 외부에 정보를 안전하게 제공한다는 특징이 있어 네트워크 보안에서 많이 활용된다. 즉, 원래 값을 숨긴 채 같은지 확인 할 수 있으며 해시 값만 보고 원래 값으로 복원하기는 어렵다.
	- 단, 암호화는 원칙적으로 복호화가 가능하지만, 해시는 복호화가 불가능하므로 암호화와는 다른 개념이다.
- 찾고자 하는 값을 O(1) 으로 바로 찾음: 키 자체가 해시 함수에 의해 값이 있는 인덱스가 되므로 값을 찾기 위한 탐색 과정이 필요 없다.
- 값을 인덱스로 활용하려면 적절한 변환 과정을 거쳐야 한다

###### 해시의 장점과 간단한 개념
- 해시를 사용하면 순차 탐색할 필요 없이 해시 함수를 활용해서 특정 값이 있는 위치를 바로 찾을 수 있어 탐색 효율이 좋다.
- 해시 테이블은 키와 대응한 값이 저장되어있는 공간
- 해시 테이블의 각 데이터를 버킷 이라고 부른다.

###### 해시의 특성을 활용하는 분야
- 특정 데이터를 탐색하는 횟수가 많을 경우 해시를 고려하는 것이 좋다.
- 비밀번호 관리 : 사용자의 비밀번호를 그대로 노출해 저장하는 것은 위험하므로 해시 함수를 활용해 해싱한 비밀번호를 저장한다. 또한, 비밀번호가 맞는지 확인할 때도 사용자가 입력한 비밀번호를 해싱해 확인한다.
- 데이터베이스 인덱싱: 데이터베이스에 저장된 데이터를 효율적으로 검색할 때 해시를 활용한다.
- 블록체인 : 블록체인에서 해시 함수는 핵심 역할을 한다. 각 블록은 이전 블록의 해시값을 포함하고 있으며 데이터 무결성을 확인할 수 있다.

##### 해시 함수
- 자바에서 해시셋 또는 해시맵이라는 표준 API 를 제공하는데 이 클래스는 해시와 거의 동일하게 동작하므로 해시를 쉽게 사용할 수 있다.
- 하지만 해시의 원리를 이해하면 좋다. 

###### 해시 함수를 구현할 때 고려할 내용
- 해시 함수가 변환한 값은 인덱스로 활용해야 하므로 해시 테이블의 크기를 넘으면 안된다.
- 해시 함수가 변환한 값의 충돌은 최대한 적게 발생해야 한다.
	- 충돌 : 서로 다른 두 키에 대해 해싱 함수를 적용한 결과가 동일한 것을 의미한다.

###### 나눗셈법

```
h(x) = x mod k
```

	- 이 때 , x가 3의 배수인 경우 규칙적으로 같은 해시 값이 반복된다.
![[Pasted image 20260525201258.png]]
- 나눗셈 법의 해시 테이블 크기는 자연스럽게 k 가 된다. k 에 대해 모듈러 연산을 했을 때 나올 수 있는 값은 0 ~ (k - 1)이기 때문이다.

###### 곱셈법
- 곱셈법은 모듈러 연산을 활용하지만 소수는 활용하지 않는다.
```
h(x) = (((x * A) mod 1) * m)
```
- m 은 최대 버킷의 개수, A 는 황금비(1.1618...)이다.
- mod 1 라는 말은 정수를 버리고 소수만 취하겠다는 뜻이다.
- 과정
	1. 키에 황금 비를 곱한다
	2. 정수 부분을 버리고 소수 부분만 취하기 위해여 1로 모듈러 연산을 한다
	3. 위에서 구한 소수 값을 가지고 실제 해시 테이블에 매핑한다. 테이블 크기가 m 이면 구한 수에 m 을 곱한 후 정수 부분을 취하는 연산을 통하면 0~(m-1)해시 테이블에 매핑할 수 있다.

##### 충돌 처리
- 서로 다른 키에 대해 해시 함수의 결괏값이 같으면 충돌이라고 한다.
- 하나의 버킷에 2개의 값을 넣을 수는 없으므로 반드시 충돌 처리를 해야 한다.

###### 체이닝으로 처리하기
- 충돌이 발생하면 해당 버킷에 LinkedList 로 같은 해시값을 가지는 데이터를 연결한다.
- 단, 충돌이 많아지면 그만큼 LinkedList 의 길이가 길어지고 다른 해시 테이블의 공간은 덜 사용하므로 공간 활용성이 떨어진다.
- 충돌이 많으면 LinkedList 자체의 한계 때문에 검색 성능이 떨어진다. LinkedList 로 연결한 값을 찾으려면 맨 앞 데이터부터 검색해야 하기 때문이다. 만약 N개의 키가 있고 모든 키가 충돌하여 체이닝 되었다면 마지막 버킷을 검색하는 경우 시간 복잡도는 O(N)이다.
- 참고로 Java 에서 HashMap 클래스는 체이닝을 사용하여 해시 충돌을 처리한다. 단, LinkedList 로 연결하는 데이터가 일정 갯수로 늘어나면 이진 탐색 트리로 변환하여 데이터 접근에 O(log N) 성능이 나오도록 개선한다.

##### 몸풀기 문제
###### 문제 18 두 개의 수로 특정 값 만들기
- 저자 권장 시간: 30분
- 권장 시간 복잡도: O(N + K)
- 출제: 저자 출제
- 문제 설명 : n개의 양의 정수로 이루어진 배열 `arr`와 정수 `target`이 주어졌을 때, 배열 안에서 **서로 다른 두 수의 합이 `target`이 되는 경우가 존재하면 `true`**, 존재하지 않으면 `false`를 반환하는 `solution()` 함수를 작성하세요.
- 제약조건
	- `n`은 2 이상 10,000 이하의 자연수입니다.
	- `arr`의 각 원소는 1 이상 10,000 이하의 자연수입니다.
	- `arr`의 원소 중 중복되는 원소는 없습니다.
	- `target`은 1 이상 20,000 이하의 자연수입니다.
- 입출력 예
	 ![[Pasted image 20260525233826.png]]
- 의사코드
	1. arr 만큼 Map 에 넣는데, 키는 해당 value 를 넣고 값은 target - 현재값 을 넣는다.
	2. key 를 집어와서 value 가 있는지 확인하고 있으면 중간에 true 를 반환한다.
	3. 전부 조사했는데도 없으면 false 를 반환한다.
- 푼 코드
``` java
class Solution {  
    public boolean solution(int []arr, int target){  
        // 1. arr 만큼 Map 에 넣는데, 키는 해당 value 를 넣고 값은 target - 현재값 을 넣는다.  
        Map<Integer, Integer> map = new HashMap<>();  
        Arrays.stream(arr).forEach(num -> map.put(num, target - num));  
  
        // 2. key 를 집어와서 value 가 있는지 확인하고 있으면 중간에 true 를 반환한다.  
        for(int key : map.keySet()){  
            if(key != map.get(key) && map.containsKey(map.get(key))){  
                return true;  
            }  
        }  
  
        // 3. 전부 조사했는데도 없으면 false 를 반환한다.  
        return false;  
    }  
}
```

###### 문제 19 완주하지 못한 선수
- [문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/42576)
- 의사 코드
	1. 참여자를 모두 HashMap 에 넣는다. 키는 이름, 값은 해당 이름을 가진 사람 수로 한다.
	2. completion 만큼 HashMap 에 있는 사람의 count 를 깎되, 0이 되면 HashMap 에서 제거한다.
	3. 마지막으로 남은 사람을 return 한다.
- 푼 코드
``` java
import java.util.*;

class Solution {
    public String solution(String[] participant, String[] completion) {
        // 참여자를 모두 HashMap 에 넣는다. 키는 이름, 값은 해당 이름을 가진 사람 수로 한다.
        HashMap<String, Integer> map = new HashMap<>();
        Arrays.stream(participant).forEach(name -> {
            if(map.containsKey(name)) {
                map.put(name, map.get(name) + 1);
            }else{
                map.put(name, 1);
            }
        });

        Arrays.stream(completion).forEach(name -> {
            if(map.containsKey(name)) {
                map.put(name, map.get(name) - 1);
            }
            if(map.get(name) == 0) {
                map.remove(name);
            }
        });

        return map.keySet().stream().map(i -> i).toArray()[0].toString();
    }
}
```
