###### 문제 01. 배열 정렬하기
- 권장 시간 10분
- 권장 시간 복잡도 O(NlogN)
- 문제 : 정수 배열을 정렬해서 반환하는 solution() 함수를 완성하세요
- 제약 조건
	- 정수 배열의 길이는 2이상 10^5 이하이다.
	- 정수 배열의 각 데이터 값은 -100, 000 이상 100,000 이하이다.
	- 입출력의 예
		- \[1, -5, 2, 4, 3] > \[-5, 1, 2, 3, 4]
		- \[2, 1, 1, 3, 2, 5, 4] > \[1, 1, 2, 2, 3, 4, 5]
		- \[6, 1, 7] > \[1, 6, 7]
- 의사 코드
	1. 배열을 정렬한다.
- 정답 코드
``` java
private static int[] solution(int[] arr){
	Arrays.sort(arr);
	return arr;
}
```

- 참고차료 
	- Arrays.sort 란?
		- 배열을 정렬하는 Java 기본 도구로 배열 타입에 따라 내부 알고리즘이 다르다.
	- 사전 지식
		- 퀵 정렬
			- 퀵 정렬(QuickSort) : 이 피벗을 중심으로 전체 배열을 더 작은 부분으로 쪼개나가는 **분할 정복(Divide and Conquer)** 알고리즘
			    - 피벗(pivot): 정렬을 수행하기 위해 기준점으로 삼는 중식축을 의미한다.
			    - 피벗을 하나 고르고 나면 알고리즘은 배열을 다음과 같은 방식으로 재배치 한다.
			        1. **피벗보다 작은 데이터**는 모두 피벗의 **왼쪽**으로
			        2. **피벗보다 큰 데이터**는 모두 피벗의 **오른쪽**으로
			        - `[피벗보다 작은 값들]` ← **피벗(Pivot)** → `[피벗보다 큰 값들]`
			    - 이 과정을 거치고 나면, 다른 데이터들은 아직 완벽히 정렬되지 않았더라도 **피벗의 위치만큼은 최종 정렬된 위치로 확정**된다.
			    - 그 후 왼쪽 그룹과 오른쪽 그룹에서 각각 새로운 피벗을 뽑아 이 과정을 반복(재귀)하면서 전체를 정렬하게 된다.
			- 피벗을 고르는 절대 규칙은 없으나 아래와 같은 방법을 사용한다.
			    - **맨 앞이나 맨 뒤 원소 선택:** 가장 직관적이고 구현이 쉽지만, 배열이 이미 정렬되어 있는 최악의 경우 성능이 크게 떨어질 수 있다.
			    - **중앙 원소 선택:** 배열의 가운데 위치한 값을 고른다.
			    - **랜덤 선택:** 배열 중 무작위로 하나를 골라 최악의 상황을 피한다.
			    - **Median-of-Three (삼분할 중간값):** 맨 앞, 맨 뒤, 중간 위치의 세 값 중 '중간값'을 피벗으로 선택하는 방법으로, 가장 효율적이라 현업에서 자주 쓰인다.
			- 퀵 정렬의 시간 복잡도는 피벗을 어떻게 고르느냐와 데이터 초기 상태에 따라 달라진다.
			    - 평균적인 경우 (Average Case) & 최선의 경우 (Best Case)
			        - 시간 복잡도: O(n log n)
			        - **상황:** 피벗을 고를 때마다 배열이 대략 **반씩(50:50) 균등하게 쪼개지는 경우**
			        - 배열을 반씩 쪼개는 단계(트리의 깊이)가 log n번 일어나고, 각 단계마다 모든 데이터(n개)를 피벗과 비교하므로 두 값을 곱한 O(n log n)이 됩니다. 대부분의 무작위 데이터에서는 이 효율적인 속도가 나온다.
			    - 최악의 경우
			        - 시간 복잡도 : O(n^2)
			        - 상황: 피벗을 골랐는데 하필 **가장 작거나 가장 큰 값**이 뽑혀서, 배열이 한쪽으로만 치우치게 쪼개지는 경우입니다. (예: 이미 오름차순 정렬된 배열에서 맨 앞 원소를 피벗으로 고를 때)
			        - 데이터가 n > n -1 > n - 2개로 하나씩만 줄어들기 때문에 쪼개는 단계가 n번 필요하게 됩니다. 결국 n단계 동안 각각 n번씩 비교하므로 O(n^2)이 되어 성능이 뚝 떨어진다.
	- int[], long[], double[] 같은 primitive 배열
		- primitive 배열은 내부적으로 보통 Dual-Pivot QuickSort를 사용한다.
		- 평균 `O(n log n)`, 최악 `O(n^2)`,  보통 `O(log n)`
		- Dual-Pivot QuickSort란?
			- 일반적인 QuerySort 는 Pivot 을 1개 고르지만, Dual-Pivot QuickSort 는 pivot 을 2개 고른다.
				- 일반적인 QuickSort: `[pivot 보다 작은 값]` `[pivot]` `[pivot 보다 큰 값]`
			- Dual-Pivot QuickSort는 pivot을 2개 고른다.
				- `[pivot1보다 작은 값]` `[pivot1~pivot2 사이 값]` `[pivot2보다 큰 값]`
				- 즉, 배열을 3구역으로 나눠서 정렬한다.
	- Integer[], String[], 객체 배열
		- 객체 배열은 내부적으로 TImeSort 를 사용한다
			- Time Sort 란?
				-  **Merge Sort + Insertion Sort + 이미 정렬된 구간 활용**이 섞인 알고리즘
				- 실제 데이터는 완전 랜덤이 아니라, 어느 정도 정렬된 구간이 있는 경우가 많은데 이런 구간을 run 이라고 부른다. 그 run 들을 효율적으로 병합해서 전체 정렬을 만든다.
				- 또한, 객체 배열 정렬을 stable sort 로 같은 값의 기존 순서가 유지된다.
				- 평균 `O(n log n)`, 최악 `O(n log n)`,  공간 복잡도(추가 필요 메모리) `O(n)`,  이미 정렬됨 `O(n)`
				- (TimeSort 는 딱 봐도 너무 어려워보여서 이 정도로 일단 스킵)
	- Primitive 는 QuickSort, 객체는 TimeSort 를 쓰는 이유
		- Primitive 는 값 자체만 있으므로 기존 순서를 유지할 필요가 없다. 그래서 빠르고 메모리를 적게 쓰는 QuickSort 계열을 쓴다.
		- 객체 배열은 정렬 기준이 다양하므로 기존 순서를 유지하기 위해 stable 한 TimeSort 를 쓴다.
	- Array.sort() 와 Collections.sort() 의 차이
		- 내부적으로 Collections.sort() 도 결국 List 를 배열처럼 다루면서 TimeSort 계열로 정렬한다. 그래서 요즘에는 `list.sort(Comparator.naturalOrder());` 로 많이 사용한다.
		- 참고로 primitive int 는 바로 내림차순을 못쓰므로 Compator를 이용한다.

###### 문제 02. 배열 제어하기
- 권장 시간 : 10분 
- 권장 시간 복잡도 O(NlogN)
- 문제 : 정수 배열을 하나 받습니다. 배열의 중복값을 제거하고 배열 데이터를 내림차순으로 정렬해서 반환하는 solution 함수를 구현하세요. 
- 제약 조건 
	- 배열의 길이는 2이상 1,000 이하입니다.
	- 각 배열의 데이터 값은 -100,000 이상 100,000 이하입니다.
	- 입출력의 예
		- \[4, 2, 2, 1, 3, 4] > \[4, 3, 2, 1]
		- \[2, 1, 1, 3, 2, 5, 4] > \[5, 4, 3, 2, 1]
- 의사 코드
	1. TreeSet 을 내림차순 용으로 만든다.
	2. 해당 컬렉션에 배열원소들을 전부 집어 넣는다.
	3. TreeSet 을 출력한다.
- 내 알고리즘의 시간 복잡도 : O(n log n)
- 푼 코드
``` java
class Solution{
	public static void solution(int[] arr){
	    Set<Integer> set = new TreeSet<>(Comparator.reverseOrder());
	    Arrays.stream(arr).forEach(set::add);
	    System.out.println(set);
	}
}
```

###### 문제03 두 개 뽑아서 더하기(프로그래머스 제공)
- [프로그래머스 문제 링크](https://programmers.co.kr/learn/courses/30/lessons/68644)
- 의사코드
	1. TreeSet 오름차순으로 만든다.
	2. 앞에서 1개를 뽑는다.
	3. 그 뒤에서 1개를 뽑는다.
	4. 두 수를 더한다.
- 예상 시간 복잡도 : O(n^2 log n)
- [푼 링크](https://github.com/l830l/Algorithms/tree/main/%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%A8%B8%EC%8A%A4/1/68644.%E2%80%85%EB%91%90%E2%80%85%EA%B0%9C%E2%80%85%EB%BD%91%EC%95%84%EC%84%9C%E2%80%85%EB%8D%94%ED%95%98%EA%B8%B0)
``` java
import java.util.*;

class Solution {
    public int[] solution(int[] numbers) {
        Set<Integer> answer = new TreeSet<>();

        for(int i = 0; i < numbers.length - 1; i++){
            for(int j = i + 1; j < numbers.length; j++){
                answer.add(numbers[i] + numbers[j]);
            }
        }

        return answer.stream().mapToInt(i -> i).toArray();
    }
}
```

- 참고 자료
	- TreeSet 과 HashSet의 차이
		- HashSet → 빠른 중복 검사, TreeSet → 정렬까지 유지하는 중복 검사
		- 중복 제거 등만 필요하거나 존재 여부만 검사하면 HashSet이 압도적으로 빠르다
		- 자동 정렬이 필요하면 TreeSet 이 좋다.
		- 단, TreeSet 은 매 삽입마다 정렬을 유지해야하여 O(n log n) 이 걸리는 반면, HashSet은 삽입마다 정렬을 유지하지 않아도 되서 O(n)의 시간이 걸려서 HashSet이 빠를 때가 많다. 즉, 매번 방 청소를 하는 것 vs 마지막에 한번 대청소 하는 방식의 차이이다. 즉, HashSet을 쓰되, 정렬된 상태가 계속 유지되어야 할 때 TreeSet 으로 넘어가는 것이 좋다.
###### 문제04 모의고사(프로그래머스 제공)
- [프로그래머스 문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/42840)
- 의사 코드
	1. 학생 각각의 클래스를 만든다.
		1. 찍는 방식을 넣는다
		2. 답변으로 제출할 번호를 넣는다.
		3. 몇 번 맞췄는지 카운트를 넣는다
	2. 전체 정답 배열만큼 반복하여 학생별로 문제를 맞췄으면 해당 학생의 카운트를 +1 을 해준다.
	3. 학생 별로 카운트 가장 높은 수를 구한다.
	4. 학생들 중 해당 카운트를 가진 학생들을 구하고 해당 학생의 번호를 뽑아 제출한다.
- 예상 시간 복잡도 : O(n)
- [푼 링크](https://github.com/l830l/Algorithms/tree/main/%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%A8%B8%EC%8A%A4/1/42840.%E2%80%85%EB%AA%A8%EC%9D%98%EA%B3%A0%EC%82%AC)
``` java
import java.util.*;

abstract class Student{
    int index;
    int count;
    int[] answer;
    int answerLength;

    Student(int index, int count, int[] answer, int answerLength){
        this.index = index;
        this.count = count;
        this.answer = answer;
        this.answerLength = answerLength;
    }
}
class A extends Student{
    A(){
        super(1, 0, new int[]{1,2,3,4,5}, 5);
    }
}
class B extends Student{
    B(){
        super(2, 0, new int[]{2,1,2,3,2,4,2,5}, 8);
    }
}

class C extends Student{
    C(){
        super(3, 0, new int[]{3, 3, 1, 1, 2, 2, 4, 4, 5, 5}, 10);
    }
}

class Solution {
    public int[] solution(int[] answers) {
        Student[] students = {new A(), new B(), new C()};
        int answerLength = answers.length;
        for(int i = 0; i < answerLength; i++){
            for(Student student : students){
                if(student.answer[i % student.answerLength] == answers[i]) student.count++;
            }
        }

        int maxCount = Arrays.stream(students)
                .max(Comparator.comparing(student -> student.count))
                .get().count;

        return Arrays.stream(students)
                .filter(student -> student.count == maxCount)
                .mapToInt(student -> student.index)
                .toArray();
    }
}
```

###### 문제05 행렬의 곱셈(프로그래머스 제공)
- [프로그래머스 문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/12949)
- 의사코드
	1. 앞의 행, 뒤의 열 = 앞의 행, 뒤의 열을 구한다.
	2. 정답으로 할 행렬을 선언하는데, 행은 앞 행렬의 행, 열은 뒤 행렬의 열로 설정한다.
	3. 앞 행렬의 열이 증가하며, 뒤의 행이 증가하며 서로 곱해지고 계속 더한다.
	4. 이것을 앞 행렬의 행과 뒤 행렬의 열의 수만큼 반복한다.
- 예상 시간 복잡도 O(n^3)
- [푼 링크](https://github.com/l830l/Algorithms/tree/main/%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%A8%B8%EC%8A%A4/2/12949.%E2%80%85%ED%96%89%EB%A0%AC%EC%9D%98%E2%80%85%EA%B3%B1%EC%85%88)
``` java
import java.util.*;

class Solution {
    public int[][] solution(int[][] arr1, int[][] arr2) {
        int x = arr1.length;
        int y = arr2.length;
        int z = arr2[0].length;
        int [][] answer = new int[x][z];

        for(int i = 0; i < x; i++){
            for(int j = 0; j < y; j++){
                for(int k = 0; k < z; k++){
                     answer[i][k] += arr1[i][j] * arr2[j][k];
                }
            }
        }

        return answer;
    }
}
```

###### 문제06 실패율(프로그래머스 제공)
- [프로그래머스 문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/42889)
- 의사코드
	1. N 만큼 반복한다. => N으로 range 를 통해 stream 을 만든다.
	2. 스테이지 별로 실패율을 계산한다.
	3. stage만큼 반복하여 실패율을 계산한다.
		- 실패율 = 클리어 못한 사람수 / 도달한 사람수
	4. 스테이지들의 실패율에 따른 스테이지를 정렬한다.
- 예상 시간 복잡도 : O(NM + NlogN)
``` java
import java.util.*;
import java.util.stream.IntStream;

class Stage {
    int stage;
    double failRate;

    Stage(int stage, double failRate) {
        this.stage = stage;
        this.failRate = failRate;
    }
}
class Solution {
    public int[] solution(int N, int[] stages) {
        // N 만큼 반복한다. => N으로 range 를 통해 stream 을 만든다.
        // 스테이지들의 실패율에 따른 스테이지를 정렬한다.
        return IntStream.range(1, N + 1)
                .mapToObj(stage -> new Stage(stage, getFailRate(stage, stages)))
                .sorted(
                        Comparator.comparing((Stage s) -> s.failRate)
                        .reversed()
                        .thenComparing(s -> s.stage)
                )
                .mapToInt(stage -> stage.stage)
                .toArray();
    }

    // 실패율 계산 함수
    public double getFailRate(int baseStage, int[] stages) {
        // 스테이지 별로 실패율을 계산한다.(아래 함수 작성)
        // stage만큼 반복하여 filter 를 통해 실패율을 계산한다.
        // 실패율 = 클리어 못한 사람수 / 도달한 사람수

        long tryPeopleNums = Arrays.stream(stages)
                .filter(stage -> stage >= baseStage)
                .count();

        if(tryPeopleNums == 0) return 0;

        long failPeopleNums = Arrays.stream(stages)
                .filter(stage -> stage == baseStage)
                .count();

        return (double)failPeopleNums / tryPeopleNums;
    }
}
```

- 개선 필요
	- 내가 푼 코드에서 시간 복잡도가 너무 오래 걸리는 문제가 있음
	- 일단, getFailRate 함수 안에서 stages[] 의 길이 만큼 2M 번 순회함
	- 또한, 바깥 스트림에서 N번 순회하므로 N * 2M 번 순회하며 정렬로 nlogn 번 걸림 따라서 O(NM + NlogN) 이 걸려버리는 문제가 있었음.
- 개선 코드 의사 코드
    1. 스테이지의 갯수를 셀 배열을 만들어준다.
    2. Stage 클래스를 만든다
        -  스테이지 번호
        -  실패율
    3. stages 를 순회하며 해당 스테이지 번호가 몇번 나왔는지 카운팅 한다.
    4. 해당 배열을 누적으로 만들어서 이미 해당 스테이지를 통과한 사람들도 카운팅을 추가로 한다.
    5. 스테이지 마다 실패율을 계산하여 스테이지에 넣어준다.
        - 실패율 계산은 클리어 못한 사람 수 / 도달한 사람 수 이다.
    6. 실패율을 기준으로 스테이지를 정렬한다.
- 개선 코드
``` java
class Stage {
    int stage;
    double failRate;

    Stage(int stage, double failRate) {
        this.stage = stage;
        this.failRate = failRate;
    }
}
class Solution {
    public int[] solution(int N, int[] stages) {
        int[] stageNums = new int[N + 2];
        ArrayList<Stage> stageList = new ArrayList<>();

        Arrays.stream(stages).forEach(stageIndex -> stageNums[stageIndex]++);
        for(int j = N; j >= 1; j--){
            stageNums[j] += stageNums[j + 1];
        }

        for(int i = 1; i <= N; i++){
            Stage current = new Stage(i, getFailRate(stageNums[i] - stageNums[i + 1], stageNums[i]));
            stageList.add(current);
        }

        return stageList.stream()
                .sorted(
                        Comparator.comparing((Stage s) -> s.failRate)
                                .reversed()
                                .thenComparing(s -> s.stage)
                )
                .mapToInt(stage -> stage.stage)
                .toArray();
    }

    // 실패율 계산 함수
    // 실패율 = 클리어 못한 사람수 / 도달한 사람수
    public double getFailRate(int failPeopleNums, int goalPeopleNums) {
        if(goalPeopleNums == 0) return 0;
        return (double)failPeopleNums / goalPeopleNums;
    }
}
```

###### 문제 07 방문길이
- [문제링크](https://school.programmers.co.kr/learn/courses/30/lessons/49994)
- 의사코드
	1. 캐릭터 좌표 x, y 를 클래스로 지정한다
	2. command 를 통해 dx 와 dy 를 구한다.
		- 단, 범위를 벗어나면 커맨드를 무시한다.
	3. x, y, dx, dy 를 클래스에 넣는다.
	4. equals 와 hashCode 를 override 하고 HashSet 에 넣는다.
	5. 그 이후 x, y 좌표를 옮긴다.
	6. HashSet 에 있는 Path 의 갯수를 센다.
- 시간 복잡도: O(N)
- 푼 코드 
``` java
class Path{  
    int beforeX;  
    int beforeY;  
    int afterX;  
    int afterY;  
  
    Path(int beforeX, int beforeY, int afterX, int afterY){  
        this.beforeX = beforeX;  
        this.beforeY = beforeY;  
        this.afterX = afterX;  
        this.afterY = afterY;  
    }  
  
    @Override  
    public boolean equals(Object obj) {  
        Path path = (Path)obj;  
        boolean conditionEqauls = path.beforeX == this.beforeX  
                && path.beforeY == this.beforeY  
                && path.afterX == this.afterX  
                && path.afterY == this.afterY;  
        boolean conditionReverse = path.afterX == this.beforeX  
                && path.afterY == this.beforeY  
                && path.beforeX == this.afterX  
                && path.beforeY == this.afterY;  
  
        return conditionEqauls || conditionReverse;  
    }  
  
    @Override  
    public int hashCode() {  
        return Objects.hash(beforeX + afterX, beforeY + afterY);  
    }  
}  
class Solution {  
    public int solution(String dirs) {  
        Set<Path> set = new HashSet<>();  
  
        // 1. 캐릭터 좌표 x, y를 클래스로 지정한다.  
        int x = 0;  
        int y = 0;  
  
  
        // 2. command 을 통해 dx 와 dy 를 구한다.  
        for(int i = 0; i < dirs.length(); i++){  
            int dx = x;  
            int dy = y;  
  
            char command = dirs.charAt(i);  
  
            // 단, 범위를 벗어나면 해당은 무시한다.  
            if(x <= -5 && command == 'L') continue;  
            if(x >= 5 && command == 'R') continue;  
            if(y >= 5 && command == 'U') continue;  
            if(y <= -5 && command == 'D') continue;  
  
            switch(command){  
                case 'U' -> dy++;  
                case 'D' -> dy--;  
                case 'L' -> dx--;  
                case 'R' -> dx++;  
            }  
            // 3. x, y, dx, dy 를 클래스에 넣는다.  
            Path newPath = new Path(x, y, dx, dy);  
  
            // 4. eqauls과 hashCode를 override 하고, HashSet 에 넣는다.  
            set.add(newPath);  
  
            // 5. 그 이후 x, y 좌표를 옮긴다.  
            x = dx;  
            y = dy;  
        }  
  
  
        // 5. count 를 센다.  
       return set.size();  
    }  
}
```
